# Изменения в формате CSV и поддержке неквадратных матриц

## Изменения в формате CSV

### Старый формат:
```
size,cpu_time_ms,gpu_naive_time_ms,gpu_shared_time_ms,speedup_naive,speedup_shared,correct
256,0.151050,4.682150,0.377890,0.03,0.40,0
```

### Новый формат:
```
rows,cols,cpu_time_ms,gpu_naive_time_ms,gpu_shared_time_ms,speedup_naive,speedup_shared,correct
256,256,0.151050,4.682150,0.377890,0.03,0.40,1
256,512,0.302100,0.721710,0.452340,0.42,0.67,1
```

**Преимущества:**
- ✅ Каждое значение в отдельной ячейке (разделитель - запятая)
- ✅ Отдельные колонки для `rows` и `cols` (удобно для фильтрации в Excel)
- ✅ Поддержка неквадратных матриц
- ✅ Более читаемый формат

## Поддержка неквадратных матриц

### Тестовые матрицы:

**Квадратные:**
- 256×256, 512×512, 1024×1024, 2048×2048, 4096×4096, 8192×8192

**Неквадратные:**
- 256×512 (узкая)
- 512×256 (широкая)
- 1024×2048 (узкая)
- 2048×1024 (широкая)
- 512×1024 (узкая)
- 1024×512 (широкая)

### Изменения в коде:

1. **main.c:**
   - Массив `test_matrices[][2]` вместо `sizes[]`
   - Отдельные переменные `rows` и `cols`
   - Обновлен заголовок CSV
   - Обновлен вывод в консоль

2. **plot_results.py:**
   - Разделение на квадратные и неквадратные матрицы
   - Два графика: один для квадратных, один для неквадратных
   - Для неквадратных используется общее количество элементов (rows × cols)

## Использование

### Запуск программы:
```bash
.\lab3_transpose.exe
```

### Результаты:
- Консольный вывод с форматом "Rows x Cols"
- CSV файл `results.csv` с новым форматом
- Каждая строка содержит полную информацию о тесте

### Построение графиков:
```bash
python plot_results.py
```

Создаст график с двумя подграфиками:
- Левый: квадратные матрицы (по размеру N)
- Правый: неквадратные матрицы (по количеству элементов)
